# price_up.md — как поднять стоимость проекта в 2–3 раза

Документ описывает, что нужно добавить к текущему MVP сайта Rutega, чтобы продать его как **B2B Customer Portal** для интернет-провайдера, работающего исключительно с юридическими лицами.

Базовая оценка текущего состояния: **0.7 – 1.2 млн ₽**.
Цель: **2.5 – 6.5 млн ₽** (в 2–6× раз дороже в зависимости от тира).

---

## Что покупает B2B-провайдер

Юрлица не покупают «красивый сайт». Они покупают **операционный инструмент**, который сокращает их издержки на работу с оператором связи: ЭДО вместо бумаги, автоматический биллинг, SLA с компенсациями, единая точка для всех офисов холдинга. Сайт + ЛК — это лишь обёртка вокруг этих процессов.

Поэтому ценообразование идёт не от «количества страниц», а от **количества автоматизированных бизнес-процессов** провайдера и его клиентов.

---

## Tier 1 — Must-have для B2B-only (без этого продать как B2B нельзя)

| Блок | Что делает | Прирост цены |
|---|---|---|
| **Контрагенты вместо пользователей** | Юрлицо = `Organization` (ИНН/КПП/ОГРН/реквизиты) с N сотрудниками. Роли: директор, бухгалтер, технарь, наблюдатель — у каждого свои разрешения в ЛК | +250–400 К ₽ |
| **ЭДО-интеграция** (Диадок / СБИС / Контур) | Договоры, счета, акты, УПД отправляются напрямую в учётную систему клиента. Это то, ради чего юрлица меняют провайдера | +400–700 К ₽ |
| **1С-интеграция** | Выгрузка платежей, синхронизация контрагентов, автоматическое выставление счетов через `1С:ERP`/`1С:Бухгалтерия` (или REST-API под Webhook) | +300–500 К ₽ |
| **Реальный биллинг** (не read-only) | Лицевой счёт юрлица, баланс, автоматическое начисление по тарифам, постоплата/предоплата, кредитный лимит, генерация PDF-счёта на каждый период | +400–600 К ₽ |
| **SLA-движок** | Публичный SLA на каждую услугу + автоматический расчёт компенсаций при простое (downtime > X часов = Y% возврата). Без этого B2B-договор не подпишут | +200–350 К ₽ |
| **Заявка на подключение с тех. обследованием** | Пин на карте → проверка тех. возможности → автоматический КП с расчётом → подпись через ЭДО. Сейчас это просто `Lead` без следующих шагов | +300–500 К ₽ |

**Подытог Tier 1: +1.85 – 3.05 млн ₽**

### Технические детали реализации Tier 1

- Расширить `prisma/schema.prisma`: новая модель `Organization` (ИНН, КПП, ОГРН, юр. адрес, реквизиты), связь `User <-> Organization` через `OrganizationMember` с полем `role`.
- Поверх текущего `getCurrentUser()` ввести `getCurrentOrganization()` и middleware-проверку прав внутри ЛК.
- Биллинг: модели `Account` (лицевой счёт), `Charge` (начисление), `Payment` (платёж), `Invoice` (счёт). Заменить read-only `Bill` на полноценный движок с шедулером (cron/BullMQ).
- ЭДО: абстрактный интерфейс `EdoProvider` с реализациями для Диадок, СБИС, Контур — у каждого провайдера разные API, но интерфейс един.
- 1С: REST-обмен через стандартный `1С:Шина` или прямой OData-интерфейс к базе.
- SLA: модель `ServiceIncident` + cron, считающий простой за период и формирующий `Compensation` к следующему счёту.

## Tier 2 — Сильная дифференциация (поднимает цену в 2–3 раза от базы)

| Блок | Что делает | Прирост цены |
|---|---|---|
| **Status Page + инциденты** | Публичная страница состояния сети (как status.cloudflare.com), история инцидентов с RFO, подписка на уведомления. Юрлица проверяют это до покупки | +150–250 К ₽ |
| **Тикет-система с SLA по категориям** | Inbox обращений с приоритетами, SLA на ответ/решение, эскалация, история по компании, не по юзеру | +250–400 К ₽ |
| **Personal Account Manager в ЛК** | У каждой компании закреплён менеджер с фото/телефоном/чатом прямо в кабинете. Админка распределяет компании между менеджерами | +150–250 К ₽ |
| **Конфигуратор корпоративных услуг** | L2-VPN между офисами, выделенная линия, MPLS, IP-транзит, аренда подсети /29-/24, хостинг в ЦОД, резервный канал — с интерактивным расчётом цены | +300–500 К ₽ |
| **Multi-site management** | Холдинг с 50 офисами видит все точки подключения на карте, статус каждой, единый счёт или раздельные ЛС | +250–400 К ₽ |
| **API для клиентов** | OpenAPI-спека: получение баланса, статусов услуг, выгрузка трафика, открытие тикетов. Крупные клиенты подключают это к своему мониторингу | +200–350 К ₽ |
| **Аудит-лог + 152-ФЗ/КИИ-готовность** | Логирование всех действий в админке и ЛК, экспорт логов, политика хранения, согласия. Для КИИ-субъектов это блокер | +150–250 К ₽ |
| **2FA / SSO** (SAML, ЕСИА для гос-сегмента) | Юрлица с ИБ-политикой не пустят сотрудников в ЛК с одним паролем | +150–250 К ₽ |

**Подытог Tier 2: +1.6 – 2.65 млн ₽**

## Tier 3 — Премиум-фичи, оправдывающие верхнюю границу

| Блок | Что делает | Прирост цены |
|---|---|---|
| **Тендерный модуль** | Приём заявок на участие в закупках 44-ФЗ/223-ФЗ, генерация типовых документов | +150–300 К ₽ |
| **Маркетплейс доп. услуг** | антиDDoS, антивирус, облачная АТС, видеонаблюдение — с реферальными партнёрами | +200–400 К ₽ |
| **Графики потребления** | Трафик/доступность по точкам, экспорт в Excel/CSV для бухгалтерии | +100–200 К ₽ |
| **NPS / CSI замеры** | В ЛК + отчёты для руководства провайдера | +80–150 К ₽ |
| **White-label режим** | Перепродажа платформы небольшим операторам | +200–400 К ₽ |

**Подытог Tier 3: +730 К – 1.45 млн ₽**

## Технический базис, который придётся обновить

То, что покупатель-провайдер потребует «по умолчанию» для прод-нагрузки на юрлица:

| Сейчас | Должно быть | Усилие |
|---|---|---|
| SQLite | PostgreSQL + миграции, бэкапы, реплика | 30–50 ч |
| Сид из `src/data/*.ts` | Headless CMS (Strapi/Directus) для контент-команды | 40–60 ч |
| In-memory rate-limit | Redis + распределённый rate-limit | 15–25 ч |
| Нет очередей | BullMQ / Temporal для отправки в ЭДО, генерации PDF, рассылок | 30–50 ч |
| Нет файлового хранилища | S3-совместимое (MinIO / Yandex Object Storage) для договоров и актов | 20–30 ч |
| Yandex.Metrika | + Sentry, OpenTelemetry, Prometheus, alerting в Telegram/PagerDuty | 30–50 ч |
| Нет CI | GitHub Actions / GitLab CI: lint → test → build → deploy с миграциями | 20–30 ч |
| Smoke E2E на 2 страницы | Покрытие критичных flow LK + админки, ≥70% юнит-покрытия | 60–100 ч |
| Один runtime-инстанс | Docker Compose с nginx + 2 app-репликами + PG + Redis + воркер | 20–30 ч |
| Нет email-сервиса | Транзакционный email (SES / SendPulse), HTML-шаблоны, unsubscribe | 20–30 ч |

**Технический апгрейд: +600 К – 1.1 млн ₽**

## Итоговая математика по ценнику

| Конфигурация | Что включено | Цена |
|---|---|---|
| **Базовый MVP** | текущее состояние репо | **0.7 – 1.2 млн ₽** |
| **B2B-Ready** | + весь Tier 1 + базовый тех-апгрейд (PG, S3, очереди) | **2.5 – 3.5 млн ₽** |
| **B2B Pro** | + весь Tier 1 + Tier 2 + полный тех-апгрейд + CI + мониторинг + 3 мес. сопровождения | **4.5 – 6.5 млн ₽** |
| **B2B Enterprise** | + Tier 3 + интеграции с ЕСИА/госуслугами + white-label + обучение их команды | **7 – 10 млн ₽** |

## Стратегия продажи

1. **Не продавайте «MVP» — продавайте roadmap.** Покажите три тира и скажите: «вот база за 1 млн, вот B2B-Ready за 3 млн, через 2 месяца. Что выбираете?» Покупатель сам выберет средний вариант.
2. **Главные крючки в продаже B2B-провайдеру:** ЭДО, 1С, SLA-движок, тикет-система. Эти четыре пункта — это ~70% воспринимаемой ценности B2B-портала.
3. **Не обещайте интеграции «под ключ» с конкретным 1С.** Делайте абстрактный API-слой, конкретную интеграцию оценивайте отдельным контрактом — у каждого юрлица свои настройки 1С.
4. **Compliance-история — отдельный аргумент.** «Архитектура учитывает 152-ФЗ и подходит для КИИ-субъектов» добавляет ~15% к цене у госконтрактных провайдеров без единой строки кода.
5. **Самое выгодное по марже:** API + SSO + status page. Делается за 2–3 недели, продаётся как «enterprise-grade», поднимает чек на 500–700 К ₽.

---

## Рекомендуемый минимальный путь

Не пытайтесь сразу строить «B2B Pro» спекулятивно. Доделайте **Tier 1** (ЭДО + 1С + биллинг + SLA + контрагенты + заявки с тех. обследованием) — это превратит MVP в продукт за **2.5 – 3.5 млн ₽** при ~250–350 часах работы. Это лучший ROI по соотношению «вложенные часы / прирост цены».

Дальнейшие тиры добавляйте только после того, как появится конкретный покупатель с конкретными требованиями — иначе риск построить не то, что нужно рынку.
